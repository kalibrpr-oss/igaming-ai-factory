# Якорь проекта: iGaming AI-Factory

Документ-опора: что это за продукт, как устроен репозиторий, что уже сделано, как запускать, куда смотреть при доработках. Обновляй вручную или попроси ассистента синхронизировать после крупных изменений.

---

## 1. Продукт в одном абзаце

**SaaS-конвейер** для генерации SEO-текстов под нишу онлайн-казино: клиент задаёт бренд, ключи, LSI, SEO-параметры и **Brand Voice**; после оплаты (в планах) система генерирует текст через **Claude (Anthropic)**, собирает **.docx** и отправляет на email. Сейчас в коде есть **фронт (Next.js)**, **бэкенд (FastAPI)**, **регистрация/заказы/ценообразование**, **демо переписывания текста** под голос (нужен ключ API и запущенный бэкенд).

## 2. Технологический стек

| Слой | Технологии |

## 4. Фронтенд (`apps/web`)

### Назначение папок (`src/`)

- **`app/`** — маршруты: `/`, `/login`, `/register`, `/dashboard`, `/order`, `/terms`, `/guide`.
- **`components/`** — UI, лейаут, маркетинг, фон.
- **`features/order-form/`** — форма заказа (ТЗ + SEO).
- **`features/generator/prompts/`** — текстовые заготовки (напр. `crypto_punk.ts`) для будущего генератора.
- **`api/`** — клиент к бэкенду (`client.ts`, `endpoints.ts`).
- **`lib/`** — утилиты, motion-пресеты, схемы заказа, `auth-storage`.
- **`styles/`** — `globals.css` (Liquid Glass, градиенты заголовков).
- **`types/`** — DTO под API.

### Дизайн (кратко)

- **Pitch black + изумруд (`gem-*`)**, эффект **Liquid Glass** (классы `liquid-glass`, `liquid-glass-hover`).
- **`MeshBackground`** — анимированный mesh-фон (Framer Motion).
- **`GradientHeading`** — переливающийся градиент на заголовках.
- Порт разработки зафиксирован: **`3333`** (см. `package.json` → `dev`).

### Переменные окружения (фронт)

Создай при необходимости **`apps/web/.env.local`**:

| Переменная | Назначение |
|------------|------------|
| `NEXT_PUBLIC_API_URL` | URL бэкенда, по умолчанию `http://127.0.0.1:8000` |

| `NEXT_PUBLIC_TELEGRAM_URL` | Ссылка на TG (в футере); дефолт в коде — `https://t.me/SetGiftSup` |

### Запуск фронта

1. Закрой старое чёрное окно с предыдущим `npm run dev`, если оно ещё открыто.
2. Дважды **`start-site.bat`** в корне `iGaming Project` (ставит зависимости при первом запуске и запускает dev).
3. В браузере: **`http://localhost:3333`**.

Задача в Cursor: **Terminal → Run Task → «Сайт: npm run dev»** (см. `.vscode/tasks.json`).

### Доступ с телефона в одной Wi‑Fi

Сейчас dev слушает по умолчанию localhost. Чтобы открыть с телефона по IP `192.168.x.x`, в скрипте `dev` можно добавить **`-H 0.0.0.0`** (и открыть брандмауэр для порта 3333). Пока не внесено в репозиторий — при необходимости попроси правку.

## 5. Бэкенд (`backend/`)

### Назначение

- REST API под префиксом **`/api/v1`**.
- **PostgreSQL** — пользователи, заказы, платежи (модели готовы; оплаты и вебхуки — в планах).
- При старте приложения вызывается **`init_db()`** — создание таблиц (для прод лучше перейти на **Alembic**).

### Переменные (`backend/.env`, скопировать из корневого `.env.example`)

| Переменная | Назначение |
|------------|------------|
| `DATABASE_URL` | `postgresql+asyncpg://...` |
| `JWT_SECRET` | Секрет подписи JWT |
| `ANTHROPIC_API_KEY` | Ключ для Claude (демо «Применить стиль» и будущая генерация) |
| `PRICE_PER_100_WORDS_CENTS` | Цена в копейках за блок 100 слов |

### Основные эндпоинты

| Метод | Путь | Назначение |
|-------|------|------------|
| POST | `/api/v1/auth/register` | Регистрация |
| POST | `/api/v1/auth/login` | JWT |
| GET | `/api/v1/auth/me` | Текущий пользователь (Bearer) |
| GET | `/api/v1/brand-voices` | Список голосов |
| POST | `/api/v1/orders/preview-price` | Превью цены |
| POST | `/api/v1/orders` | Создать заказ (Bearer) |
| GET | `/api/v1/orders` | Список заказов |
| GET | `/api/v1/orders/{id}` | Один заказ |
| POST | `/api/v1/demo/rewrite` | Демо: переписать текст под `voice_id` (нужен ключ Anthropic) |
| GET | `/api/v1/admin/users` | Админ-список (нужен `is_admin`) |

### Brand Voice (бэкенд)

- `backend/app/prompts/registry.py` — реестр.
- Голоса: **`bold_daring`**, **`expert_precise`**, **`friendly_warm`** (модули в `prompts/voices/`).

### Запуск бэкенда (локально)

Из папки **`backend`** (после `pip install -r requirements.txt`):

```bash
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

CORS в `main.py` разрешает localhost на типичных портах фронта, включая **3333**.

## 6. Связка фронт ↔ бэк

- Фронт ходит на **`NEXT_PUBLIC_API_URL`** + пути `/api/v1/...`.
- Токен JWT хранится в **`localStorage`** (`access_token`), подставляется в `Authorization: Bearer` в `api/client.ts`.
- Демо переписывания: **`DemoSection`** → `demoRewrite()` → **`POST /api/v1/demo/rewrite`**.

## 7. Что уже сделано vs что в планах

**Сделано (по коду):**

- Лендинг, Terms, Guide, регистрация/логин, кабинет со списком заказов, форма заказа с SEO-полями и превью цены.
- Визуал: mesh, glass, изумрудная тема, модалка теста стиля.
- Бэк: пользователи, заказы, расчёт цены, JWT, админ-список пользователей, демо-rewrite через Claude.
- Документация SEO-полей: `Docs/seo-order-spec.md`.

**В планах (типичный roadmap):**

- ЮMoney / СБП, вебхуки, смена статуса заказа.
- Очередь (Redis + worker), полная генерация статьи, DOCX, почта (Resend/SendGrid).
- Alembic, прод-деплой, домен, HTTPS.

## 8. Текущий этап (синхрон с `ГДЕ_Я_ОСТАНОВИЛСЯ.txt`)

На момент последнего обновления якоря:

- Сайт смотри через **`start-site.bat`** → **`http://localhost:3333`**.
- Ключ **Anthropic** и постоянный запуск бэкенда могли быть **отложены**; без них кнопка **«Применить стиль»** не вызовет ИИ.

Правило: **перед новым запуском фронта закрой старое чёрное окно** с `npm run dev`, чтобы не плодить процессы и не путать порты.

## 9. Куда класть новые фичи (соглашение)

- Новый UI-фичи: **`apps/web/src/features/<имя>/`**.
- Новые API-роуты бэка: **`backend/app/api/v1/`**, бизнес-логика — **`services/`**.
- Новый Brand Voice: файл в **`prompts/voices/`** + строка в **`registry.py`**.
- Промпты для фронтового генератора (если отдельно от бэка): **`apps/web/src/features/generator/`**.
