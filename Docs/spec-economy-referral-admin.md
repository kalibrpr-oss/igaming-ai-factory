# Утверждённая спецификация: Credits, рефералка, админка

**Статус:** зафиксировано владельцем (источник правды — обсуждение, блок «второе сообщение» по рефу и ВВ).  
**Вывод средств:** не делаем — только внутренние Credits.  
**Пароли в админке:** не показываем (только сброс/блокировка при необходимости).

---

## Внутренняя валюта (Credits)

- Название в интерфейсе: **Credits** (при желании позже бренд: Conveer Credits).
- Списание только на заказы генерации; пополнение — рублями/криптой через платёжку → зачисление Credits.
- Комбинированная оплата заказа: Credits + доплата с баланса после пополнения — по мере реализации продукта.
- **Скидка 30% на первый заказ** — явно на страницах входа/регистрации и в оферте/подсказках к заказу.

---

## Реферальная система (канон)

- Любой зарегистрированный пользователь может приглашать рефералов.
- В ЛК: реферальная ссылка; **один раз** — кастомный slug в URL, если свободен (эстетика вместо случайной строки).
- Привязка реферала: **первый заход по ссылке** → cookie/серверное правило; после регистрации смена «чужой ссылки» реферера не меняет (уточнять в FAQ одной фразой).
- **Начисления только после успешного пополнения реферала (Credits на балансе реферала):**
  - **Первое успешное пополнение** конкретного реферала: реферер получает **50%** от суммы **этого** пополнения (**единоразово на этого реферала**).
  - **Каждое следующее пополнение** этого же реферала: реферер получает **15%** от суммы пополнения (**бессрочно**).
- Начисления **мгновенно** после подтверждения платежа (webhook, идемпотентность).
- Антиабуз: реф ≠ сам себе, логи по платежам, лимиты/флаги по мере необходимости.

## Панель владельца (админка)

Обязательно:

- Список пользователей: кто, когда зарегистрировался.
- Карточка пользователя: email, username; заказы — краткая сводка.
- Платежи: отдельная лента/страница — когда, сумма, метод пополнения.
- Рефералы: кого пригласил, когда, сколько всего получил с рефералов.
- **Последний заход / активность:** время последнего визита; по возможности **тип устройства** (телефон / ПК) из User-Agent (точность «онлайн сейчас» — ориентировочно, не как мессенджер).
- Бан пользователя без обхода регистрации на тот же email; выдача Credits / бесплатных генераций по username с причиной в логе.
- Доступ только владельцу: сильная авторизация, 2FA, по возможности ограничение по IP/VPN.

Дополнительно (из предыдущего обсуждения): поиск, фильтры, дашборд, экспорт, аудит действий админа.

## Порядок внедрения (рекомендуемый)

1. **Генерация по заказу** — стабильный путь: оплаченный/допустимый баланс → LLM → статусы → без «лживых» ошибок в UI.
2. **Пополнение Credits (ЮKassa/ЮMoney)** — тестовый контур, webhook, журнал платежей, зачисление.
3. **Реф-привязка при регистрации** — cookie + `referrer_user_id`, генерация/кастом `referral_code`, ссылки в ЛК.
4. **Начисления рефу 50% / 15%** — отдельные проводки в ledger, идемпотентность по payment id.
5. **Скидка 30% на первый заказ** — правило в pricing + тексты на регистрации/входе.
6. **Расширение админки** — таблицы, карточка пользователя, бан, выдача бонусов, **last_seen + user-agent → тип устройства**.
7. **Публичные тексты** — Credits, рефералка, скидка; без противоречий с цифрами.
8. **Домен/хостинг** — когда будешь готов; деплой через git/CI, не ручная заливка.

Параллельно можно: копирайт Conveer, формулировка «iGaming + другие темы», логотип.
