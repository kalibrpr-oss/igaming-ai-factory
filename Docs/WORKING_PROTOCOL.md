# Протокол работ (безопасность изменений)

## Резервные копии и откат
- **Предпочтительно:** Git в корне проекта (`iGaming Project`), коммит после каждого логически завершённого шага.
- **Сейчас:** если Git не установлен — поставь [Git for Windows](https://git-scm.com/download/win), затем в корне проекта: `git init`, `.gitignore` (уже есть или добавим), первый коммит «baseline».
- **Запасной вариант:** перед крупным этапом — копия всей папки проекта в архив с датой в имени (вручную).

## Правила разработки
- Один смысловой шаг → один коммит (когда Git есть).
- Перед изменениями: `pytest` по backend (минимум существующие тесты).
- После изменений: те же тесты + ручная проверка затронутого сценария.
- Не смешивать в одном PR: рефералку и оплату и админку — только очередной пункт дорожной карты.
- Если **5–10** попыток не решают одну и ту же ошибку — фиксируем симптомы, логи, **радикальная альтернатива** (другой подход, упрощение scope, откат коммита).

## Источник правды по продукту
- `Docs/spec-economy-referral-admin.md` — экономика, реф, админка.
- `Docs/development-roadmap.md` — порядок этапов и чеклисты.

## Что «не ломать»
- Регистрация / вход / verify без явной задачи на изменение.
- Платежи и баланс — только с тестовым контуром и идемпотентностью.
